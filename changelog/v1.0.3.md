# v1.0.3 — 2026-07-20

### Added
- **`project_context(operation="architecture")`**: Exposes a detailed architecture map of the project, including technology stack frameworks, languages, directory-based module layout, entry point files/methods, and core class hubs based on CodeGraph degree.
- **`task_pipeline` Integration**: `task_pipeline` now executes `get_project_architecture()` concurrently inside the `parallel_run` pool, integrating the results into the top-level `architecture_map` response.
- **Marker-based Rule Deployment**: Modified `deploy_rules.py` to support replacing wrapped rules within `<!-- agent-guidance:start -->` and `<!-- agent-guidance:end -->` blocks instead of skipping on second run. Allows updating system rules while preserving custom user rules in files.

### Changed
- Documented `architecture` operation in `server.py` and `docs/reference/mcp-surface.md`.
- Updated `docs/ARCHITECTURE.md` module map, tool list, and subsystem details to match current 9-tool architecture.

### Fixed
- Robust database error fallback inside `get_project_architecture()`. If `codegraph.db` fails to initialize or is locked, the handler degrades gracefully by setting `database_indexed: false` and returning the file-system and configuration mapping, preventing MCP timeout.

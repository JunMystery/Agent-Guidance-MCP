# Accessibility (A11Y) Checklist

This document ensures that all frontend code generated by AI is inclusive and accessible to users with disabilities.

> **When to use this skill:** `@reference` this file when the AI is asked to write or review HTML, React, Vue, or any UI components.

---

## 1. Web Content Accessibility Guidelines (WCAG)

All user interfaces **MUST** meet the [WCAG 2.1 Level AA](https://www.w3.org/WAI/WCAG21/quickref/) standard.

## 2. A11Y Implementation Rules

When generating UI code, the AI must ensure the following:

### Semantic HTML
- Always use native semantic elements (`<nav>`, `<main>`, `<article>`, `<button>`, `<a>`) over `<div>` or `<span>` where applicable.
- Do not use a `<div>` with an `onClick` handler when a `<button>` is appropriate.

### Keyboard Navigation
- All interactive elements must be fully reachable and operable via the `Tab` and `Enter`/`Space` keys.
- **Focus Management:** Active elements must have a highly visible focus indicator (`:focus-visible`). Never use `outline: none` without providing a custom visual focus state.
- Focus should be trapped inside Modals/Dialogs when they are open.

### ARIA (Accessible Rich Internet Applications)
- *First rule of ARIA: No ARIA is better than bad ARIA.* Rely on native HTML first.
- Use `aria-label` or `aria-labelledby` for buttons that only contain icons.
- Use `aria-expanded`, `aria-hidden`, and `aria-live` correctly for dynamic state changes.

### Visual & Color Contrast
- Text and interactive elements must have a contrast ratio of at least **4.5:1** against their background (3:1 for large text).
- Do not rely on color alone to convey information (e.g., error states should have an icon or text, not just red color).

### Media & Forms
- All `<img>` tags must have an `alt` attribute. If it's purely decorative, use `alt=""`.
- All `<input>`, `<select>`, and `<textarea>` elements must have an associated `<label>`.

## 3. Automated Testing

Frontend projects should integrate accessibility testing into their CI pipeline:

- Run `eslint-plugin-jsx-a11y` for React projects.
- Use `axe-core` in Cypress/Playwright integration tests to catch structural A11Y violations automatically.
